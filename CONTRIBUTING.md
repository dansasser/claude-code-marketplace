### Reporting Bugs

Bugs are tracked as [GitHub Issues](https://github.com/dansasser/claude-code-marketplace/issues). Before opening a new issue, please check to see if your bug has already been reported.

When you create a bug report, please include:

1.  **Which plugin?** (e.g., `claude-ollama-agents`)
2.  **A clear title:** "Bug: /analyze command fails on Python files".
3.  **Steps to reproduce:** Be as specific as possible.
4.  **Expected behavior:** What you expected to happen.
5.  **Actual behavior:** What actually happened.

### Suggesting Enhancements or New Plugins

I'd love to hear your ideas\! The best way to share them is by [opening a new issue](https://github.com/dansasser/claude-code-marketplace/issues).

Please provide:

1.  **A clear title:** "Feature: Add a `/lint` command to claude-ollama-agents".
2.  **A detailed description:** Explain your idea and *why* you think it would be valuable for my marketplace.
3.  **Any relevant examples:** Mockups or code snippets are always helpful.

### Submitting Pull Requests

If you're ready to contribute code to fix a bug or add a new feature, here's the workflow:

1.  **Fork** the repository.
2.  **Create a new branch** for your work: `git checkout -b feature/your-awesome-feature`.
3.  **Make your changes** and commit them with a clear message.
4.  **Push** your branch to your fork: `git push origin feature/your-awesome-feature`.
5.  **Open a Pull Request** (PR) from your fork to my `main` branch.
6.  **Describe your changes** in the PR and link to any relevant issues.

I'll review your PR as soon as I can. Thank you for helping me build this collection\!
